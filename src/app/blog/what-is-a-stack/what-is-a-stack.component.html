Definition

The Stack: is a data structure or also an abstract data type that iplements the LIFO principle.

LIFO stands for Last Input First Output; in simpler terms The last element that was pushed is the first element that comes out.

A simple analogy to grasp this concept: just think about a can of Pringles... when getting your hand in to get one chip, you are first grabbing the last chip that was put into the can.

Other example: Think about the browser history on Firefox/Chrome, etc: every time you click a link and a new page loads, the page you are leaving is inserted into the stack. So whenever you click on the back button, you are actually retieving the last element you inserted... until you reach the bottom or end of the stack.

Or the classical example... a stack of dished, from where you can only pick the dish from the top of the stack (unless you would also want to pick up a broom).

Ok now, the basic operations you can perform on a stack are inserting a new element and retrieving an element. These both basic operations are usually called push and pop respectively.

To implement these, it's all up to the programming language of your choice and it most likely the language already have this abstract data type in it's features so you just have to learn how to use it. In general, an array or list is used to implement and use a stack.

Push and pop operations are always executed over the top of the stack. This term is used to define the end of the list.

In the next section, we will review the differences of the stack implementation in a couple of programing languages and the usage of the stack.

A quick note when working with Python: If we try to pop an empty stack, we will get an exception (it's just the way Python works). Usually we will wrap the pop operation in a try catch block to avoid this exception. You could also check for the length of the stack before attempting a pop operation.

Complexity analysis (big O)

The stack is an abstract data type with tons of implementations in programming. In terms of time complexity while reading and writing is constant O(1). That is because all the operations are done in the top of the stack; we don't have to reorder or move any other elements of the stack.
