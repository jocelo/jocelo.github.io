Cuantas veces al dia necesitas entrar a sitios con casi la misma URL y que solo cambia una pequeña parte (un ID, o un parámetro) y terminas siempre entrando a una liga vieja o anterior en tu navegador y solo cambias esa pequeña parte de la URL?

El primer ejemplo que viene a la mente es el software que usamos para el Project Tracking (como Jira); en donde para poder ver bugs o stories, casi siempre es la misma URL y solo cambia el número del bug o de la story que estas trabajando.

Otro ejemplo es github, cuando trabajas en diferentes repositorios y necesitas brincar de uno a otro varias veces en un periodo corto de tiempo; o bien cuando vas a aprobar Pull Requests.

Inclusive si entre tus responsabilidades esta revisar perfiles de personas en redes sociales o de trabajo, como linkedIn, también puedes usar este tip para localizar el perfil que estas buscando mucho mas rápido.

Este proceso de "abre la pagina, cambia el pedacito, carga la pagina de nuevo" es muy recurrente en el dia a dia de un desarrollador y mi ecologista interno que desea evitar el desperdicio (del servidor, del tiempo, del teclado, de la pila de la lap... ya tu sabes) me llevo a preguntarme si habría alguna manera de evitar esta constante repetición ? Y vaya que lo hay! En este articulo vamos a revisar como puedes usar tu navegador para mejorar la manera de acceder a este tipo de URLs que denominaremos como: URL's con palabras clave.

Bien ya sabemos que tenemos que hacer, ahora el como; Depende completamente del navegador que estés usando, asi que vamos por partes.

Chrome

Version al momento de escribir esto: 98.0.4758.80

Para poder configurar chrome, necesitamos agregar un nuevo motor de búsqueda; debes hacer lo siguiente:

  1. Da click en el icono de configuración ( ) en la parte superior derecha de la ventana.
  2. Selecciona settings.
  3. En la ventana de configuración, selecciona search engine en el menu izquierdo.
  4. Expande la sección manage search engines.
  5. Da click en el botón add para que aparezca una ventana modal en la que ingresaras la configuración de cada sitio que quieras agregar.

Por ejemplo, podemos agregar el repositorio de angular en github, específicamente la URL para revisar los Pull Requests; la idea es que solamente escribiendo en el URL la palabra clave github un espacio y el numero del Pull Request, te lleve directamente a la URL correcta. La configuración para lograr esto es:

  • Search engine: https://github.com/
  • Keyword: ghpr
  • URL with %s in place of query: https://github.com/angular/angular/pull/%s

Nota: La keyword de esta configuración es una abreviación de github pull request; puedes poner la palabra clave que te haga mas sentido a ti y claro, tu propio repositorio.

Guardando los cambios, es momento de probar nuestra configuración: Abre una pestaña nueva en Chrome y en la barra de navegación tecleamos github 45269 y al dar enter, observaremos la URL se convierte a https://github.com/angular/angular/pull/45269... excelente!

Otro ejemplo, podemos configurar la URL para MDN WebDocs (previamente conocido como Mozilla Developer Network) y realizar búsquedas directamente sobre su sitio web y no tener que pasar primero por algún buscador. Para esto la configuración que agregaremos es la siguiente:

  • Search engine: https://developer.mozilla.org
  • Keyword: mdn
  • URL with %s in place of query: https://developer.mozilla.org/en-US/search?q=%s

Para usarlo, solo debes escribir mdn array foreach

Si trabajas con FontAwesome, seguro seguido entras a su pagina web para buscar un icono en especifico. Aquí te dejo la configuración que necesitas para optimizar esta búsqueda:

  • Search engine: https://fontawesome.com/
  • Keyword: fa
  • URL with %s in place of query: https://fontawesome.com/search?q=%s&s=solid

Ahora ya solo deberás escribir fa angular para ver directamente el icono de angular.

Firefox

Version al momento de escribir esto: 97.0.2

En el caso de Firefox, el proceso es un poco mas sencillo; solo debes agregar un nuevo marcador (bookmark):

  1. Damos click en el menu de configuración ( ).
  2. Selecciona bookmarks y Manage bookmarks.
  3. Dentro de la ventana de librería de marcadores solo debes dar click en la opción de Organize y Add Bookmark.

Ahora tomaremos como ejemplo (entre mis favoritos) a Jira, para este caso los datos que debes ingresar son:

  • Name: https://yourproject.atlassian.net
  • URL: https://yourproject.atlassian.net/browse/BUG-%s
  • keyword: bug

Nota: Deberás cambiar la parte de la URL que hace referencia a tu propio proyecto, y es muy probable que en la configuración de Jira, a los bugs se les ponga el prefijo bug. Si este no es tu caso, también deberás modificar la URL del ejemplo para que refleje el prefijo correcto.

Brave

Version al momento de escribir esto: 1.36.111
  1. Da click en el icono de configuración ( ) en la parte superior derecha de la ventana.
  2. Selecciona settings.
  3. En la ventana de configuración y menu izquierdo selecciona search engine.
  4. Expande la sección other search engines.
  5. Da click en el botón add para que aparezca una ventana modal en la que ingresaras la configuración de cada sitio que quieras agregar.

Los campos que debes llenar son similares a los de Chrome (ya que Brave se basa en Chromium).

Conclusión

Como puedes observar, las partes mas importantes son los keywords y reemplazar la parte dinámica de la URL con el símbolo %s. Puedes reemplazar la parte dinámica en cualquier parte de la URL, no solo el final.

Espero este articulo haya sido de tu agrado. Estoy seguro que ya pensaste en mas sitios donde puedes aplicar este pequeño pero poderoso hack de productividad, solo tienes que encontrar la mejor forma de segmentar las URL's para insertar el símbolo y obtener el máximo beneficio.
