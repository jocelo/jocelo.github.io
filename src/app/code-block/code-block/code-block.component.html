  3. If the current character is an opening bracket, we push it to the stack.
  4. If the current character is a closing bracket, we pop an element from the stack.

  6. If we are done processing the string, but we have at least one element remaining in the stack, it means there's at least one opening bracket that does not have their corresponding pair.
