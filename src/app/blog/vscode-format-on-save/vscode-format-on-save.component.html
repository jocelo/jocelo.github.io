Formatting source files with VSCode

The following was tested using VSCode version 1.64.1.

Do you know about the super useful "Format on Save" settings on VSCode? With it, you can automatically apply the default formats and prettifiers that come with VSCode as soon as you save a file.

It's a pretty handy tool that used correctly, can save you a bunch of time. It's like wearing the same outfit everyday for work... you can worry about more important things and leave the mundane things to others.

To enable it, you just have to:

  1. Click on Preferences > Settings
  2. Type in the search bar "formatonsave" and click the checkmark

And viola! from this moment on, whenever you work on a file and save it, you will get the swift hand of VSCode massaging your code to make it look pretty!!

But wait, there's more...

VSCode Default Formatters

By default, VSCode have installed formatters for many programming languages. If you want to take a look at all the default formatters and support that VScode has, you can go click on extensions, on the left menu, click the filter icon on the top of the screen (next to EXTENSIONS) and type "language".

For some languages you can edit some of the settings directly on the settings screen. Just open the settings screen (same way we did above) and scroll down to the extension options, expand it and search for whatever file settings you want to configure.

i.e. You can edit HTML formatting options to include a new line before the end of file; define column width after line break (default to 120 characters); enable/disable documentation of HTML tags while hovering over a tag; and more cool stuff!

Exclude files from autoFormat

Ok now we are cruising at full speed formatting our files on save but... suddenly you noticed that your JSON files are also being formatted. Maybe you do not want to format JSON files (in one of my projects, this was a curse rather than a blessing) or another type of file. You can exclude files from the reach of the magic wand of VSCode. You just have to disable the auto format for that specific file, follow this steps:

  1. Open command pallete:
    • mac: Cmd + Shift + P
    • linux: Ctrl + Shift + P
    • win: Ctrl + Shift + P
  2. Type "configure language specific settings" > select (or search) JSON file type > press enter.

    With this the settings.json file will open on edit mode, and the cursor will be located inside a section with a tag for the language you selected.

  3. You just have to disable the formatonsave option, adding this: "editor.formatOnSave": false

That's all you have to do... now JSON files will not be auto-formatted when saved.

Of course you can do things the opposite way... disable formatonsave by default, and just enable it for whatever file you want, following the steps above for each file language as needed.
