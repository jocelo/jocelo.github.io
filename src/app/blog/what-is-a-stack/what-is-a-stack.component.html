Definition

The Stack: is a data structure or also an abstract data type that implements the LIFO principle.

LIFO stands for Last Input First Output; in simpler terms The last element that was pushed is the first element that comes out.

A simple analogy to grasp this concept: just think about a can of Pringles... when getting your hand in to get one chip, you are first grabbing the last chip that was put into the can.

Other example: Think about the browser history on Firefox/Chrome, etc: every time you click a link and a new page loads, the page you are leaving is inserted into the stack. So whenever you click on the back button, you are actually retrieving the last element you inserted... until you reach the bottom or end of the stack.

Or the classical example... a stack of dishes, from where you can only pick the dish from the top of the stack (unless you would also want to pick up a broom).

Ok now that we understand what is a stack, the basic operations you can perform on a stack are inserting a new element and retrieving one element. These two basic operations are usually called push and pop respectively.

To implement these, it's all up to the programming language of your choice and its most likely the language already has this abstract data type included and ready to be used in it's core features so you just have to learn how to use it. In general, an array or list is used to implement a stack.

Push and pop operations are always executed on the top of the stack; This term is used to define the end of the list.

Complexity analysis (big O)

The stack is an abstract data type with tons of different use cases. In terms of time complexity while reading and writing is constant or O(1). That is because all the operations are done at the top of the stack; we don't have to reorder or move any other elements of the stack while performing push and pop operations.
